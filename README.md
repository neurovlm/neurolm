# NeuroLM (Static + WASM)

This project is now **WASM-only at runtime**.

- Inference runs in-browser via Rust/WASM (`neurolm/wasm` + `neurolm/site/wasm-worker.js`).
- The frontend does **not** require `/api/*` Python endpoints.

## Model Card Note

The `Qwen3-0.6B-pubmed-neuroimaging-dapt` model was trained on approximately **1.2 million PubMed publications** related to **cognitive neuroscience**.

## Required static files

Serve these files from your site:

- `site/index.html`
- `site/styles.css`
- `site/app.js`
- `site/engine.js`
- `site/wasm-worker.js`
- `site/pkg/*` (generated by `wasm-pack`)
- model directory with:
  - `config.json`
  - `generation_config.json`
  - `tokenizer.json`
  - `model-q4_k_m.gguf` (or `model-q4_0.gguf` / `model.gguf`)

Model lookup order is automatic:

1. `./model`
2. `../model`

So either `site/model/*` or `model/*` (sibling to `site`) works.

## Convert GPTQ safetensors -> Candle GGUF

Input model:

- `/home/rph/efficient_ai/project/Qwen3-0.6B-pubmed-neuroimaging-dapt-gptq-w4a16`

Conversion pipeline:

1. GPTQ -> dequantized HF (`tools/export_gptq_to_hf.py`)
2. HF -> GGUF + quantize (`tools/hf_to_gguf.sh`, via llama.cpp)

If `llama.cpp` is not installed:

```bash
git clone https://github.com/ggerganov/llama.cpp.git ~/llama.cpp
cmake -S ~/llama.cpp -B ~/llama.cpp/build
cmake --build ~/llama.cpp/build -j --target llama-quantize
```

One-command wrapper:

```bash
cd /home/rph/efficient_ai/neurolm
./tools/convert_gptq_to_candle_gguf.sh \
  /home/rph/efficient_ai/project/Qwen3-0.6B-pubmed-neuroimaging-dapt-gptq-w4a16 \
  /tmp/qwen3-hf \
  /home/rph/efficient_ai/neurolm/model \
  q4_k_m
```

This produces (at minimum):

- `neurolm/model/model-q4_k_m.gguf`
- `neurolm/model/config.json`
- `neurolm/model/generation_config.json`
- `neurolm/model/tokenizer.json`

By default, intermediate `model-f16.gguf` is deleted after quantization. Set
`KEEP_F16_GGUF=1` before running if you want to keep it.

